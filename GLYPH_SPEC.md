# AXN Glyph Mapping Specification

## Overview

The Alexanarch Identifier (AXN) maps content hashes to emoji sequences using a fixed, published 256-glyph table. Each byte value (0x00–0xFF) corresponds to exactly one emoji. The mapping is deterministic: the same content always produces the same emoji identifier.

## Format

```
AXN:FAMILY.EMOJI₁EMOJI₂EMOJI₃EMOJI₄
```

- **AXN** — Alexanarch namespace
- **FAMILY** — Sémantique Potentielle mint family (semantic domain)
- **EMOJI₁₋₄** — First 4 bytes of SHA-256, each mapped to one glyph

## Verification

Anyone can verify an AXN:
1. Compute SHA-256 of the original content
2. Take the first 4 bytes
3. Map each byte to its emoji using the published table
4. Compare to the claimed AXN

## Properties

- **Deterministic:** Same content → same identifier, always
- **Verifiable:** Anyone with the content can recompute and verify
- **Irrevocable:** Content-derived, no authority can revoke
- **Human-readable:** 4 emoji are memorable and citable
- **Collision-resistant:** 2³² (4B) possibilities in 4-emoji form; extendable to 6 emoji (2⁴⁸)
- **Semantically navigable:** Cluster names provide visual navigation

## Relationship to the Glyphic Checksum

The AXN is the **deterministic address**. The Glyphic Checksum (GCS) is the **non-deterministic structural signature** — generated by an AI system translating the structural arc of the content into emoji. The AXN tells you WHAT the work is. The GCS tells you what the work DOES.

Both travel with the deposit. Only the AXN is used for citation.
