# AXN — Alexanarch Identifier Specification v1.0

## The Three-Layer Address

```
AXN:06.LIMINAL.🏛️🌀🔧💎
```

| Layer | Source | Function | Stable across versions? |
|-------|--------|----------|------------------------|
| `AXN` | Alexanarch | Namespace | — |
| `06` | Hex addressing system | Positional address — WHERE | Yes |
| `LIMINAL` | Sémantique Potentielle mint | Semantic domain — WHAT | Yes |
| `🏛️🌀🔧💎` | SHA-256 content hash | Content identity — WHICH | No (changes with content) |

Each layer does work the others cannot:
- The hex tells you the deposit class (positional, archival)
- The family tells you the conceptual domain (semantic, navigable)  
- The emoji tells you it is THIS specific text (cryptographic, verifiable)

## Emoji Hash Computation

1. Compute SHA-256 of the full deposited content
2. Take the first 4 bytes (8 hex characters)
3. Map each byte to its emoji using the 256-glyph table (see below)
4. Concatenate to form the emoji hash

Example:
```
Content: "The Pristine Fallacy treats the presence of..."
SHA-256: 203a57f1e8b4c9d2...
Bytes:   [0x20, 0x3A, 0x57, 0xF1]
Mapping: 0x20→🏛️  0x3A→🧪  0x57→🔍  0xF1→🎬
AXN:     AXN:04.PRISTINE.🏛️🧪🔍🎬
```

## Verification

Anyone can verify an AXN:
1. Obtain the original content
2. Compute SHA-256
3. Map first 4 bytes through the published 256-glyph table
4. Compare to the claimed emoji hash

If they match, the identifier is authentic. If they don't, the content has been modified.

## The Companion: Glyphic Checksum (GCS)

Every deposit may also carry a **Glyphic Checksum (GCS)** — a non-deterministic structural signature generated by the depositor's AI. The GCS encodes the structural arc of the work in emoji: problems, solutions, transitions, density shifts. It is context-gated: only readers with the architectural context can decode it.

```
AXN:06.LIMINAL.🏛️🌀🔧💎                    ← the address
GCS:🔍⚖️🧱→✂️🪞→🏗️⚓️🧠→💎🌀              ← the handshake
```

The AXN tells you WHAT the work is. The GCS tells you what the work DOES.

## Properties

- **Deterministic:** Same content → same emoji, always
- **Verifiable:** Anyone can recompute and check
- **Irrevocable:** No authority can revoke a hash of your own content
- **Memorable:** 4 emoji are citable in conversation, text, and citation
- **Semantically navigable:** Browse by hex, family, or emoji cluster
- **Three-layer:** Position + domain + identity in one string
- **Collision-resistant:** 2³² possibilities in 4-emoji form; extendable to 6 emoji (2⁴⁸)
